# 디자인 시스템

Toss 스타일을 참고한 미니멀 & 화이트 노트 앱의 디자인 시스템.
**모든 스타일 작업 전 해당 파일을 `@` 참조한다.**

---

## 파일 구성

| 파일                               | 내용                                    | 언제 참조                           |
| ---------------------------------- | --------------------------------------- | ----------------------------------- |
| [`tokens.md`](./tokens.md)         | 컬러·라운드·그림자·간격 CSS 변수 전체   | 색상 선택, CSS 변수 사용, 경계 표현 |
| [`typography.md`](./typography.md) | 폰트 크기·굵기·행간·자간 스케일         | 텍스트 스타일 작업                  |
| [`layout.md`](./layout.md)         | 반응형 메이슨리 그리드, 상단바 규칙     | 레이아웃·그리드 구현                |
| [`components.md`](./components.md) | 버튼·카드·인풋·모달·토스트·빈 상태 스펙 | UI 컴포넌트 구현·수정               |
| [`motion.md`](./motion.md)         | 트랜지션·애니메이션·아이콘·단축키       | 애니메이션, 아이콘 추가             |

---

## 설계 원칙

1. **미니멀이 기본** — 요소는 제 역할을 할 때만 등장한다. 장식보다 여백.
2. **부드러운 피드백** — 모든 상태 변화는 짧고 자연스러운 트랜지션(`cubic-bezier(0.16, 1, 0.3, 1)`)으로 전달한다.
3. **선이 아니라 면과 그림자** — 경계를 테두리 대신 옅은 그림자와 톤 차이로 표현한다.
4. **하나의 강조색** — 블루는 액션·강조에만 쓴다. 본문은 무채색 위계로 정리한다.
5. **둥근 형태** — 카드 20px, 버튼/인풋 14px의 일관된 라운드로 친근한 인상을 만든다.
