# 디자인 토큰

컬러·라운드·그림자·간격의 전체 CSS 변수 정의.
CSS 변수 사용, 색상 선택, 경계 표현 작업 시 이 파일을 참조한다.

> **참고** — 아래 토큰은 설계 기준값이며 `src/index.css`의 Tailwind `@theme` 변수와 별도로 정렬이 필요하다.

## Do / Don't

**Do**

- **모든 색상은 반드시 CSS 변수(`var(--)`)로만 사용한다.** hex·rgb·rgba·hsl 값을 코드에 직접 쓰지 않는다.
- 필요한 색상이 토큰에 없으면 `tokens.md`에 먼저 추가한 뒤 사용하고, 추가 사실을 반드시 알린다.
- 블루(`--blue`)는 버튼·링크·포커스·고정 마크 등 **액션과 강조에만** 사용한다.
- 채도는 `--blue` / `--red` 계열에만 허용한다. 새 색이 필요하면 기존 토큰의 명도만 조정해 파생한다.
- 텍스트 위계는 `--ink` → `--ink-2` → `--ink-3` 순서로만 낮춘다.
- 경계 표현은 반드시 `--shadow-sm/md/lg` 그림자와 배경 톤 차이로 한다.
- 컴포넌트 라운드는 용도에 맞는 토큰을 사용한다 (카드 20px, 버튼 14px, 아이콘 버튼 11px).
- 그림자 단계는 기본 → hover → 모달 순으로 강도를 높인다.

**Don't**

- hex(`#rrggbb`), `rgb()`, `rgba()`, `hsl()` 색상값을 코드에 직접 사용하지 않는다 — 반드시 CSS 변수를 통해 사용한다.
- 토큰에 없는 색상이 필요할 때 코드에 바로 하드코딩하지 않는다 — 토큰 추가 후 변수로 사용한다.
- 디자인 시스템에 없는 임의 색상(초록, 노랑, 보라 등)을 추가하지 않는다.
- 일반 본문이나 배경에 블루 계열을 사용하지 않는다.
- `--ink`와 `--ink-3`을 같은 위계에서 병용하지 않는다.
- 카드나 인풋에 `border: 1px solid`를 사용해 경계를 표현하지 않는다 (핀 상태 예외).
- 그림자값을 하드코딩하지 않는다 — 반드시 토큰 변수를 사용한다.
- 라운드 값을 임의로 조합하지 않는다 (예: 카드에 8px, 버튼에 24px).
