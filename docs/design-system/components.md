# 컴포넌트

버튼·카드·인풋·모달·토스트·빈 상태의 스펙 정의.
UI 컴포넌트 구현·수정 시 이 파일을 참조한다.
토큰 값은 `tokens.md` 참조.

---

## 버튼

| 종류    | 클래스              | 높이    | 배경       | 텍스트    | 특이사항                                                       |
| ------- | ------------------- | ------- | ---------- | --------- | -------------------------------------------------------------- |
| Primary | `.btn-primary`      | 46–48px | `--blue`   | 흰색      | active: `scale(0.96)`, shadow: 블루                            |
| Save    | `.btn-save`         | 48px    | `--blue`   | 흰색      | 내용 없으면 `disabled` (opacity 0.4)                           |
| Ghost   | `.btn-ghost`        | 48px    | `--line-2` | `--ink-2` | hover: `--line` 배경                                           |
| Icon    | `.icon-btn`         | 34×34px | 투명       | `--ink-3` | r: 11px, `.active`=`--blue-soft`, `.danger:hover`=`--red-soft` |
| Danger  | `.confirm-row .del` | 50px    | `--red`    | 흰색      | 삭제 확인 단계에서만 사용                                      |

**Do**

- 주요 액션(저장·생성)에만 Primary/Save 버튼을 사용한다.
- 아이콘 버튼은 카드 액션(핀·수정·삭제)에만 사용한다.
- 버튼 클릭 시 `scale(0.96)` active 피드백을 반드시 포함한다.

**Don't**

- 같은 영역에 Primary 버튼을 2개 이상 배치하지 않는다.
- Ghost 버튼을 주요 액션에 사용하지 않는다.
- Danger 버튼을 직접 노출하지 않는다 — 반드시 확인 단계(confirm-row) 이후에만 표시한다.

## 카드 (`.card`)

- **기본**: 흰 표면 + `--shadow-sm`, r: `--r-card`(20px)
- **hover**: `translateY(-3px)` + `--shadow-md`, transition 0.2s
- **핀 상태(`.pinned`)**: `--blue` 22% opacity 보더 + 우상단 핀 마크
- **등장**: `card-in` 애니메이션 (0.32s, `cubic-bezier(0.16, 1, 0.3, 1)`)
- **액션 버튼(핀·수정·삭제)**: hover/focus 시 fade-up, 모바일은 항상 노출

**Do**

- 카드 hover는 위로 뜨는 효과(`translateY`)와 그림자 강화를 함께 사용한다.
- 카드 등장 시 반드시 `card-in` 애니메이션을 적용한다.
- 모바일에서는 액션 버튼을 항상 노출한다 (hover 감지 불가).

**Don't**

- 카드 `border-radius`를 `--r-card`(20px) 외 다른 값으로 설정하지 않는다.
- hover 효과로 배경색만 바꾸는 단순 처리를 하지 않는다 (`translateY` + `shadow` 조합 필수).
- 핀 표시를 텍스트 라벨이나 배지로 대체하지 않는다 (우상단 핀 아이콘 마크 사용).

## 입력

| 종류                       | 높이 | 배경                 | 포커스                                  |
| -------------------------- | ---- | -------------------- | --------------------------------------- |
| 검색 `.search input`       | 46px | 흰색 + `--shadow-sm` | 블루 보더 + `--blue-soft` 4px 포커스 링 |
| 에디터 제목 `.title-input` | —    | 투명 (보더 없음)     | 포커스 링 없음                          |
| 에디터 본문 `.body-input`  | —    | 투명 (보더 없음)     | 포커스 링 없음                          |

- 검색 인풋 플레이스홀더: `--ink-3`
- 에디터 인풋 플레이스홀더: `#c5cbd1`

**Do**

- 검색 인풋은 포커스 시 블루 보더 + 4px `--blue-soft` 포커스 링을 함께 표시한다.
- 에디터 인풋은 인라인 느낌을 유지한다 (보더·배경 없음).

**Don't**

- 에디터 제목·본문 인풋에 테두리나 배경을 추가하지 않는다.
- 포커스 링을 생략하거나 브라우저 기본 outline으로 대체하지 않는다.

## 모달 (`.modal`)

- **스크림**: `rgba(15,23,42,.36)` + `backdrop-filter: blur`
- **데스크톱**: 화면 중앙, r: `24px`, 등장: `modal-in` (0.28s)
- **모바일**: 하단 시트, 상단 r `24px`, 등장: `sheet-in` (아래에서 슬라이드, 0.32s)
- **구성**: 헤더(핀 토글 + 닫기) → 바디(제목/본문) → 푸터(글자 수 + 취소/저장)

**Do**

- 모바일에서는 시트 형태로 아래에서 슬라이드 인한다.
- `Esc` 키와 스크림 클릭으로 모달을 닫는 동작을 반드시 지원한다.

**Don't**

- 데스크톱 모달을 모바일에서 그대로 중앙 팝업으로 사용하지 않는다.
- 스크림 없이 모달을 표시하지 않는다.
- 등장 애니메이션을 생략하지 않는다.

## 토스트 (`.toast`)

- **배경**: `--ink` (다크), 체크 아이콘 색: `#4cc98f`
- **위치**: 하단 중앙
- **소멸**: 2.2초 후 자동 제거
- **등장**: `toast-in` 애니메이션 (0.3s, `cubic-bezier(0.16, 1, 0.3, 1)`)

**Do**

- 저장·삭제 등 성공 피드백을 토스트로 전달한다.
- 2.2초 후 자동으로 사라지게 한다.

**Don't**

- 에러 메시지를 토스트로 사용하지 않는다 (`alert()` 도 금지 — `console.error` 사용).
- 토스트를 여러 개 동시에 쌓지 않는다.
- 토스트에 닫기 버튼을 추가하지 않는다 (자동 소멸).

## 빈 상태 (`.empty`)

두 가지 변형:

1. **메모 없음** — 아직 메모가 없을 때
2. **검색 결과 없음** — 검색어에 매칭되는 메모가 없을 때

구성: 둥근 아이콘 칩(74px) + 제목 + 안내 문구

**Do**

- 목록이 비었을 때 반드시 빈 상태 컴포넌트를 표시한다.
- 메모 없음과 검색 결과 없음을 구별해 다른 메시지를 표시한다.

**Don't**

- 빈 목록을 그냥 공백으로 두지 않는다.
- 빈 상태에서 불필요한 CTA 버튼을 추가하지 않는다 (상단 + 버튼으로 충분).
