# 태그 기능 정의서 (확정)

## 기능 개요

노트에 태그를 추가하고 관리할 수 있다.

## 데이터 구조

`Note` 타입에 `tags: string[]` 필드 추가.

```ts
type Note = {
  id: string;
  title: string;
  content: string;
  tags: string[];
  createdAt: string;
  updatedAt: string;
};
```

- 기존 노트의 `tags`가 없는 경우 빈 배열 `[]`로 처리
- 태그는 입력값 그대로 저장 (대소문자 구분 안 함 — 소문자로 정규화 후 저장)

## 기능 요구사항

### 태그 추가

- NoteEditor 내 태그 입력 필드에서 문자열 입력 후 Enter로 추가
- 추가된 태그는 chip 형태로 표시
- 중복 태그 입력 시 무시 (소문자 정규화 기준으로 비교)
- 앞뒤 공백은 자동으로 trim
- 빈 문자열, 공백만 있는 입력은 무시
- 태그 하나당 최대 20자
- 노트당 최대 10개

### 태그 삭제

- 각 chip의 X 버튼 클릭으로 삭제
- 삭제 확인 없이 즉시 제거

### 태그 표시

- NoteEditor의 제목·본문 사이에 태그 영역 배치
- NoteList의 NoteItem에는 태그 미표시 (이번 범위 외)
- 태그가 없으면 chip 묶음 영역(tag-area)을 렌더링하지 않음 (단, 첫 태그 추가용 입력 필드는 항상 노출)

## 저장 시점

- 태그는 별도로 저장하지 않음
- 노트 저장(Save) 버튼 클릭 시 title, content와 함께 tags도 함께 전송

## 엣지케이스

| 케이스                         | 처리                                                    |
| ------------------------------ | ------------------------------------------------------- |
| `React`와 `react` 동시 입력    | 소문자 정규화로 중복 처리, 두 번째 입력 무시            |
| 공백만 입력 후 Enter           | 무시                                                    |
| 20자 초과 입력                 | 입력 자체를 막음 (maxLength 속성)                       |
| 10개 초과 추가 시도            | 입력 필드 비활성화                                      |
| 태그 추가 후 저장 안 하고 이탈 | 기존 노트 선택 시 변경사항 유실 (현재 저장 정책과 동일) |
