# 검색 기능 정의서 (확정)

## 개요

노트 목록을 실시간으로 필터링하는 클라이언트 사이드 검색 기능.
검색어를 입력하면 사이드바의 노트 목록이 즉시 좁혀지고, 지우면 전체 목록으로 돌아온다.

## 핵심 시나리오

1. **title + content 동시 검색**: 검색어가 노트 제목 또는 본문 어디에든 포함되면 목록에 표시 (OR 조건)
2. **실시간 필터링**: 매 키 입력마다 즉시 필터링 (debounce 없음)
3. **검색어 초기화**: 검색어를 지우면 전체 목록으로 복귀

## 경계 조건 및 예외

| 케이스                                                                | 동작                                      |
| --------------------------------------------------------------------- | ----------------------------------------- |
| 빈 문자열                                                             | 전체 목록 표시                            |
| 공백만 입력 (`"   "`)                                                 | trim 후 빈 문자열로 처리 → 전체 목록 표시 |
| 검색 결과 0건                                                         | "검색 결과가 없습니다" 메시지 표시        |
| 대소문자                                                              | case-insensitive 비교                     |
| 검색 중 노트 선택 후 검색어 변경으로 해당 노트가 목록에서 사라진 경우 | 에디터 선택 상태 유지 (초기화 안 함)      |

## UI 배치

- 검색 입력창은 사이드바 상단, 노트 목록 위에 배치
- `Layout` 컴포넌트 수정 없음 — `sidebar` slot에 검색창 + 목록을 함께 주입

## 기술 방향

- 검색 로직은 `useSearchNotes(notes, query)` 커스텀 훅으로 분리
- 검색 상태(`query`)는 `App.tsx`에서 관리 (컴포넌트 간 공유 필요)
- 현재는 클라이언트 사이드 필터링 (`Array.filter()`) 사용
  - 추후 API 검색으로 전환 시 debounce 추가 필요

## Out of Scope (이번에 하지 않을 것)

- URL 쿼리 파라미터 연동 (`?q=검색어`)
- 서버사이드 필터링 (JSON Server `?q=` 파라미터)
- 검색 결과 하이라이팅
- 태그 검색 (tags 필드 미구현)
- 검색 히스토리

## 용어 정의

| 용어                      | 정의                                                      |
| ------------------------- | --------------------------------------------------------- |
| 검색어 (query)            | 사용자가 입력한 필터링 문자열. trim 후 사용.              |
| 필터링                    | `notes[]`를 클라이언트에서 `Array.filter()`로 좁히는 동작 |
| 검색 결과 (filteredNotes) | query를 적용한 후 남은 노트 배열                          |
