# 필터 기능 정의서 (확정)

## 기능 개요

필터로 원하는 노트만 확인할 수 있다. 태그를 기준으로 노트 목록을 필터링한다.

---

## 결정 사항 요약

### Q1. 다중 선택 시 필터 조건 — **OR 선택**

`React`와 `TypeScript`를 선택하면 둘 중 하나라도 가진 노트를 모두 표시한다.  
AND 방식은 태그를 많이 선택할수록 결과가 급격히 줄어 사용성이 떨어지므로 OR를 선택.

### Q2. 필터 UI 위치 — **사이드바 상단 (노트 목록 위)**

별도 필터 패널을 두지 않고 `NoteList` 상단에 태그 칩(chip) 목록을 배치한다.  
필터와 목록을 같은 공간에서 보여주면 결과를 즉시 확인할 수 있어 인지 비용이 낮다.

### Q3. 필터에 표시할 태그 목록 — **전체 노트의 태그 합집합**

현재 필터 결과와 관계없이 존재하는 모든 태그를 표시한다.  
필터 결과 기준으로 태그를 줄이면 선택 순서에 따라 선택지가 사라져 혼란스럽다.

### Q4. 필터 상태 유지 범위 — **세션 내 유지, 새로고침 시 초기화**

다른 노트를 선택하거나 편집 화면으로 이동해도 필터는 유지된다.  
하지만 페이지 새로고침 시에는 초기화(전체 보기)된다. LocalStorage 저장은 오버엔지니어링.

### Q5. 필터 초기화 방법 — **"전체" 칩 또는 선택 태그 재클릭**

- 선택된 태그 칩을 다시 클릭하면 해제된다.
- 아무것도 선택되지 않은 상태가 곧 "전체 보기"이며, 별도 "전체" 버튼은 두지 않는다.
- 2개 이상 선택된 경우 "필터 초기화" 버튼을 노출해 한 번에 해제할 수 있다.

### Q6. 필터 적용 중 노트 생성 — **필터 상태 무관하게 생성, 필터는 초기화하지 않음**

필터가 걸린 상태에서 새 노트를 생성해도 필터를 자동으로 변경하지 않는다.  
새 노트가 현재 필터 조건에 맞지 않으면 목록에 보이지 않을 수 있으며, 이는 의도된 동작이다.

### Q7. 일치하는 노트가 없을 때 — **빈 상태 메시지 표시**

"선택한 태그와 일치하는 노트가 없습니다." 메시지를 노트 목록 영역에 표시한다.  
아무 노트도 없는 기본 빈 상태와 구분되는 문구를 사용한다.

---

## 기능 요구사항

- 노트에 태그를 기준으로 필터 기능을 가진다.
- 태그는 여러 개 선택이 가능하다.
- 다중 선택 시 OR 조건으로 필터링한다 (선택 태그 중 하나라도 가진 노트를 표시).
- 아무것도 선택하지 않았을 경우에는 전체 노트를 보여준다.
- 필터 UI는 `NoteList` 상단에 태그 칩 형태로 배치한다.
- 태그 칩 목록은 전체 노트에 존재하는 태그의 합집합을 표시한다.
- 선택된 태그 칩을 다시 클릭하면 해제된다.
- 2개 이상 선택 시 "필터 초기화" 버튼을 노출한다.
- 필터 상태는 세션 동안 유지되며, 새로고침 시 초기화된다.
- 필터 조건에 맞는 노트가 없으면 빈 상태 메시지를 표시한다.

## 데이터 구조

```ts
// 필터 상태 (App.tsx 또는 NotesContext에서 관리)
type FilterState = {
  selectedTags: string[]; // 선택된 태그 배열, 빈 배열 = 전체 보기
};

// 필터링 로직 (OR 조건)
function filterNotes(notes: Note[], selectedTags: string[]): Note[] {
  if (selectedTags.length === 0) return notes;
  return notes.filter((note) => note.tags?.some((tag) => selectedTags.includes(tag)));
}
```

## UI 구조

```
NoteList
  ├─ TagFilter (태그 칩 목록 + 초기화 버튼)
  │    ├─ TagChip ("React") — 선택/해제 토글
  │    ├─ TagChip ("TypeScript") — 선택/해제 토글
  │    └─ [필터 초기화] — 2개 이상 선택 시 노출
  └─ 필터링된 노트 목록 or 빈 상태 메시지
```

---

## 엣지 케이스

| 상황                                  | 동작                                              |
| ------------------------------------- | ------------------------------------------------- |
| 태그가 없는 노트                      | 필터 선택 시 목록에서 제외됨                      |
| 전체 노트에 태그가 하나도 없음        | TagFilter 영역을 렌더링하지 않음                  |
| 노트 삭제 후 해당 태그가 더 이상 없음 | 태그 칩 목록에서 자동으로 제거됨                  |
| 필터 중 새 노트 생성                  | 필터 유지, 새 노트가 조건 미충족 시 목록에 미표시 |
| 선택된 태그가 있는 모든 노트 삭제     | 빈 상태 메시지 표시, 해당 태그 칩은 사라짐        |

---

## 비고

- 필터는 이후 추가될 수 있다 (날짜, 검색어 등).
- `selectedTags` 상태는 필터 결과에만 영향을 주며, 노트 데이터 자체를 변경하지 않는다.
